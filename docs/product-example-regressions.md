# Product example regression ledger

This ledger is the durable provenance record for defects discovered at the
product-example boundary. It connects the consumer-visible symptom to its owner,
focused regression, retained application journey, and fixing change. The package
tests prove the smallest owning-layer behavior; the Playwright journey proves that
the behavior still composes inside a usable application.

The ledger supplements package changelogs and pull-request descriptions. Its IDs
are permanent: do not rename or reuse an ID after it has been published.

## Entry contract

Add or update an entry whenever an example exposes a defect, including when the
defect belongs to the example rather than Octane:

1. Describe the consumer-visible symptom, not an implementation hypothesis.
2. Classify the owner as `octane`, a published binding or integration, or the
   private application/tooling.
3. Link the smallest focused behavioral regression in the owning package. For an
   application-owned defect, state that the application journey is the owning
   regression instead of inventing a package test.
4. Link the retained Playwright journey or other consumer-level gate.
5. Link the fixing pull request and merge commit. Use `this change` until those
   values exist.
6. Record whether the result is fixed, open, or an accepted temporary workaround.

An accepted temporary workaround is exceptional. Its entry must name an owner,
the upstream issue or blocked layer, the exact removal condition, and the journey
that must continue to pass after removal. A vague comment such as “needed for
Octane” is not sufficient. Framework-owned defects normally block example
graduation until the owning package and focused regression land in the same
change.
