# Octane runtime + compiler cleanup plan

> **EXECUTED 2026-07-03.** All four phases landed (5 fix agents + a foundational
> API pass; every stage gated on the octane test project, then the full monorepo:
> 1382 tests green, typecheck + format clean). Maintainer decisions applied: public
> API = React parity only (getCurrentScope/getCurrentBlock/withScope removed;
> index.ts reorganized into three commented tiers: React-parity / semi-public
> compiler-emit / test-only); clone() hydration recovery ungated to prod;
> useDeferredValue initialValue unified on transition priority; `_poc_*`/`_fold_*`
> scaffolding tests consolidated into named suites; forBlock's dead `extra` param
> removed atomically across runtime + compiler emit. 12 changesets cover the
> behavioral fixes. Two audit items intentionally kept: test-only exports
> (documented as tier 3 in index.ts) and the runtime↔compiler Capture-suffix
> parsing (different programs; no shared module possible).
> The tables below are retained as the audit record.

> Produced 2026-07-03 from a 28-agent adversarial audit (14 section/cross-cutting
> auditors, every finding re-verified by an independent skeptic; 117 confirmed, 5
> downgraded, 0 refuted) over `runtime.ts` (9,285 lines), `runtime.server.ts` (966),
> and `compiler/compile.js` (6,622). Findings below are deduplicated and phased by
> value and risk. Every Phase-1 item is a REAL behavioral bug (mostly drift between
> duplicated code paths — the strongest argument for the Phase-2 refactors).

## Execution rules

- Work the phases in order; Phase 2's shared helpers are the *vehicle* for several
  Phase-1 fixes (build the helper, fix the drift inside it, delete the copies).
- Gate every phase on: `pnpm test` (full monorepo), `pnpm typecheck`,
  `pnpm format:check`. Phase 1 items additionally each get a targeted regression test
  and a changeset; Phases 2–4 are behavior-preserving (no changesets) except where noted.
- The conformance + differential suites are the drift detectors — run
  `packages/octane/tests/differential/` and `conformance/` after each Phase-2 refactor.

## Decision points (maintainer input wanted before executing)

1. **`getCurrentScope`/`getCurrentBlock`** (runtime.ts:373) — zero consumers anywhere.
   Remove, or keep as a documented public escape hatch for external binding authors?
2. **`clone()` hydration structural detection** (runtime.ts:2429) — currently dev-only
   (gated on the compiler's dev `loc` arg), contradicting the "recovery runs in prod"
   contract every other site follows. Ungate (recommended), or document as intentional?
3. **`useDeferredValue` initialValue swap priority** (runtime.ts:7328) — urgent vs the
   steady-state transition path. Unify on transition (recommended, matches React), or
   comment the fork?
4. **`_poc_*` / `_fold_probe` / `_app_usediv` / `_return_count` scaffolding tests** —
   fold the still-valuable assertions into the real suites and delete the files, or keep?
5. **`forBlock`'s dead `extra` param** — removing changes the compiler-emitted call
   signature (compiler + runtime must move together; old compiled output in caches
   would break). Cheap now, pricier after any public release.
