# Bindings

The `@octanejs/*` packages are faithful octane ports of React libraries. Each
one mirrors its upstream's architecture: the framework-agnostic core is reused
verbatim wherever the upstream ships one, and only the thin React binding layer
is reimplemented on octane's hooks — so most code works by changing the import.

| Package                     | Ports                 | One-liner                                                                   |
| --------------------------- | --------------------- | --------------------------------------------------------------------------- |
| `@octanejs/zustand`         | zustand               | The bear-necessities state store, bound to octane's `useSyncExternalStore`. |
| `@octanejs/query`           | TanStack Query        | `useQuery`/`useSuspenseQuery` data fetching over the verbatim query-core.   |
| `@octanejs/router`          | TanStack Router       | Code-based routing with concurrent, transition-driven navigation.           |
| `@octanejs/motion`          | Motion (Framer)       | Animation primitives on motion-dom.                                         |
| `@octanejs/stylex`          | StyleX                | Compiled atomic CSS via the StyleX babel pipeline.                          |
| `@octanejs/lexical`         | @lexical/react        | The Lexical editor binding.                                                 |
| `@octanejs/floating-ui`     | @floating-ui/react    | Positioning + interactions for tooltips, popovers, menus.                   |
| `@octanejs/radix`           | Radix UI Primitives   | Accessible unstyled UI primitives.                                          |
| `@octanejs/mdx`             | @mdx-js/react         | MDX documents compiled to octane components (this page is one).             |
| `@octanejs/testing-library` | react-testing-library | `render`/`renderHook`/`act` over the verbatim @testing-library/dom.         |

Plus the metaframework plugin:

- `@octanejs/vite-plugin` — dev SSR, routing, and hydration wiring for full
  apps (this site runs on it).

## The porting recipe

Every binding follows the same plan (documented per-package in the repo):

1. **Reuse the core.** zustand's vanilla store, `@tanstack/query-core`,
   `@tanstack/router-core`, `@testing-library/dom`, `@mdx-js/mdx` — all
   framework-agnostic, all reused unmodified.
2. **Port only the React layer.** Usually a few hundred lines of hooks and
   provider components, rewritten on octane's identical hook API.
3. **Prove parity differentially.** The test suites run the same fixtures
   through the octane binding and the real React library, asserting identical
   DOM output after identical event sequences.

## Using a binding

```tsrx
import { useQuery, QueryClient, QueryClientProvider } from '@octanejs/query';

function Repos() @{
	const { data, isPending } = useQuery({
		queryKey: ['repos'],
		queryFn: () => fetch('/api/repos').then((r) => r.json()),
	});

	@if (isPending) {
		<p>Loading…</p>
	} @else {
		<ul>
			@for (const repo of data; key repo.id) {
				<li>
					{repo.name as string}
				</li>
			}
		</ul>
	}
}
```
